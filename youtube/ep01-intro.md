## 인트로 (30초)

**[화면: 터미널에서 Claude Code 실행 장면]**

> 안녕하세요. 오늘부터 Claude Code의 `.claude/` 디렉토리를 활용하는 방법을 시리즈로 소개합니다.
>
> Claude Code 쓰시는 분들, 매번 같은 설명 반복하고 계시지 않나요?
> "커밋 메시지는 이 형식으로 해줘", "테스트는 꼭 먼저 돌려줘"...
>
> `.claude/` 폴더 하나면, 이런 규칙을 **코드로 정의**해서 모든 프로젝트에서 재사용할 수 있습니다.

---

## 본론 1: .claude/ 란? (1분)

**[화면: 디렉토리 구조 그래픽]**

```
.claude/
├── settings.json       # 설정
├── commands/           # 슬래시 명령어
├── skills/             # 자동 적용 규칙
├── agents/             # AI 서브프로세스
└── hooks/              # 이벤트 스크립트
```

> `.claude/`는 프로젝트 루트에 두는 **설정 디렉토리**입니다.
>
> 여기에 파일을 추가하면 Claude Code가 자동으로 인식합니다.
> 별도 설치나 설정 없이, 파일만 넣으면 동작합니다.

## 본론 2: 5가지 구성 요소 (2분)

**[화면: 각 요소 아이콘과 설명]**

> 핵심 구성 요소 5가지를 빠르게 훑어볼게요.

### Commands

> 첫 번째, **Commands**.
> `/commit`, `/push` 같은 슬래시 명령어를 직접 만들 수 있습니다.
> Markdown 파일 하나가 명령어 하나입니다.

**[화면: /commit 실행 장면]**

### Skills

> 두 번째, **Skills**.
> Claude가 **자동으로** 적용하는 규칙입니다.
> "이미지는 상대경로로", "API 응답은 이 형식으로" 같은 규칙을 정의해두면
> Claude가 알아서 따릅니다.

### Agents

> 세 번째, **Agents**.
> 복잡한 작업을 **독립 프로세스**로 실행합니다.
> 계획 세우기, 코드 실행, 검증 같은 전문화된 작업에 사용합니다.

### Hooks

> 네 번째, **Hooks**.
> 세션 시작 같은 **이벤트에 반응**하는 스크립트입니다.
> Node.js로 작성하고, 버전 체크나 환경 확인 같은 자동화에 씁니다.

### Settings

> 마지막, **Settings**.
> 이 모든 걸 연결하는 설정 파일입니다.
> `settings.json`에서 hooks 등록하고, 상태줄 설정합니다.

## 본론 3: 왜 필요한가? (1분 30초)

**[화면: Before/After 비교]**

> 왜 이렇게 복잡하게 하냐고요?

### Before: 매번 반복

```
나: "커밋 좀 해줘"
Claude: "어떤 형식으로 할까요?"
나: "Conventional Commits로"
Claude: "알겠습니다"
... (다음 세션)
나: "커밋 좀 해줘"
Claude: "어떤 형식으로 할까요?"  ← 또 물어봄
```

### After: 한 번 정의, 계속 사용

```
나: "/commit"
Claude: [자동으로 Conventional Commits 형식으로 커밋]
```

> `/commit` 명령어에 규칙을 정의해두면, 다시는 설명할 필요 없습니다.
>
> 더 좋은 건, 이 설정을 **여러 프로젝트에서 공유**할 수 있다는 겁니다.
> Git submodule로 `.claude/`를 관리하면, 새 프로젝트에서도 바로 사용 가능합니다.

## 본론 4: 시작하기 (1분)

**[화면: 터미널에서 직접 따라하기]**

> 지금 바로 시작해볼까요?

```bash
mkdir -p .claude/commands
```

> 먼저 디렉토리를 만들고,

```bash
echo '# Test Command

테스트 명령어입니다.' > .claude/commands/test.md
```

> 간단한 명령어 파일을 만듭니다.

**[화면: Claude Code에서 /test 실행]**

> Claude Code를 실행하면, `/test` 명령어가 바로 인식됩니다.
> 이게 전부입니다. 파일 하나 만들면 명령어 하나가 생깁니다.
